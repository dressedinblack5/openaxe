Interact with Language Server Protocol (LSP) servers to get code intelligence features.

Supported operations:
- goToDefinition: Find where a symbol is defined
- findReferences: Find all references to a symbol
- hover: Get hover information (documentation, type info) for a symbol
- documentSymbol: Get all symbols (functions, classes, variables) in a document
- workspaceSymbol: List project-wide symbols matching a query string
- goToImplementation: Find implementations of an interface or abstract method
- prepareCallHierarchy: Get call hierarchy item at a position (functions/methods)
- incomingCalls: Find all functions/methods that call the function at a position
- outgoingCalls: Find all functions/methods called by the function at a position
- codeAction: Get available code actions (quick fixes, refactorings, auto-imports) at a position
- rename: Rename a symbol across the entire codebase using the language server's semantic understanding
- prepareRename: Check if a symbol at a position can be renamed, and get the range to rename
- typeDefinition: Find the type definition of a symbol (e.g., the class of a variable)
- signatureHelp: Get function signature and parameter information at a call site
- completion: Get code completion suggestions at a position
- formatting: Format a document using the language server's formatter. Returns text edits to apply.
- applyCodeAction: Find and apply a specific code action (quick fix, auto-import, refactoring) by title. First call codeAction to list available actions, then pass the title to this operation.

All operations require:
- filePath: The file to operate on (for workspaceSymbol and formatting, this is used to select the LSP server)
- line: The line number (1-based, as shown in editors) — not needed for workspaceSymbol, documentSymbol, or formatting
- character: The character offset (1-based, as shown in editors) — not needed for workspaceSymbol, documentSymbol, or formatting

Some operations also accept:
- query: A query string for workspaceSymbol. Empty string requests all symbols.
- newName: The new name for the rename operation (required for rename).
- title: The title of the code action to apply (required for applyCodeAction).
- tabSize: Tab size for formatting (default 2).
- insertSpaces: Use spaces for formatting (default true).

For workspaceSymbol, filePath is not sent in the LSP workspace/symbol request. It is used by opencode to select and start the matching LSP server.

Note: LSP servers must be configured for the file type. If no server is available, an error will be returned.
